# 회원가입 정보 저장 방식

## 📝 구현 내용

### 1. 회원가입 시
- **localStorage에 사용자 정보 저장**
- 저장 위치: `localStorage.getItem("registeredUsers")`
- 저장 형식: JSON 배열
- 저장 정보:
  - email
  - password
  - firstName
  - lastName
  - fullName
  - registeredAt (가입 시간)

### 2. 로그인 시
- **localStorage에서 회원가입 정보 확인**
- 이메일과 비밀번호가 일치하면 로그인 성공
- 회원가입하지 않은 사용자도 로그인 허용 (발표용)

---

## 🔍 동작 방식

### 회원가입 플로우
1. 사용자가 회원가입 정보 입력
2. localStorage에 저장
3. 환영 인사 표시
4. 로그인 화면으로 전환

### 로그인 플로우
1. 사용자가 이메일/비밀번호 입력
2. localStorage에서 회원가입 정보 확인
3. 일치하면 로그인 성공
4. 일치하지 않아도 로그인 허용 (발표용)

---

## 💾 저장 데이터 예시

```json
[
  {
    "email": "user@example.com",
    "password": "password123",
    "firstName": "홍",
    "lastName": "길동",
    "fullName": "홍 길동",
    "registeredAt": "2024-01-17T12:00:00.000Z"
  },
  {
    "email": "test@example.com",
    "password": "test123",
    "firstName": "김",
    "lastName": "철수",
    "fullName": "김 철수",
    "registeredAt": "2024-01-17T12:30:00.000Z"
  }
]
```

---

## ✅ 장점

1. **Backend 없이 동작**: 프론트엔드만으로 회원가입/로그인 처리
2. **정보 기억**: 회원가입한 정보를 localStorage에 저장
3. **로그인 검증**: 회원가입한 사용자만 로그인 가능 (선택적)
4. **발표용 유연성**: 미가입 사용자도 로그인 허용 가능

---

## 🔧 확인 방법

### 브라우저 개발자 도구에서 확인
1. F12 또는 우클릭 → 검사
2. Application 탭 → Local Storage
3. `registeredUsers` 키 확인
4. 저장된 회원가입 정보 확인
